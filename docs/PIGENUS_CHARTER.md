# PiGenus Charter

This document is the binding definition for PiGenus. Future implementation work must preserve this mission.

## Final Definition

PiGenus is the always-available private core instance of GENUS. It preserves information, organizes work, connects systems, and ensures long-term continuity.

Technically, PiGenus is a persistent orchestration node: a durable coordination and memory node.

## Exact Definition

PiGenus is the permanently available private infrastructure core of the GENUS system, operated on energy-efficient local hardware, primarily Raspberry Pi. It handles memory, coordination, administration, secure reachability, and cross-system orchestration.

PiGenus is not a single chatbot, not the primary language model, and not a high-performance compute machine. It is the stable base layer on which changing intelligences, workers, devices, and services collaborate.

## Five Fundamental Functions

### 1. Persistence

PiGenus preserves state over time:

- Memories
- Histories
- Configurations
- Task state
- Decisions
- Project data
- Logs
- Knowledge condensations

PiGenus ensures GENUS does not start from zero in every session.

### 2. Orchestration

PiGenus distributes work to suitable resources:

- Select workers
- Assign tasks
- Receive results
- Coordinate flows
- Manage priorities
- Use resources carefully
- Schedule time windows

PiGenus does not need to do everything itself. It must cause the right thing to happen.

### 3. Administration

PiGenus manages system operations:

- User rights
- Device trust
- System rules
- Versions
- Backups
- Status monitoring
- Error handling
- Security policies

PiGenus is the operations layer of GENUS.

### 4. Interface Readiness

PiGenus connects different devices, services, and instances:

- Phone
- Laptop
- Worker machines
- Cloud services
- APIs
- GitHub
- Future smart-home systems
- Further GENUS modules

PiGenus is the network node of GENUS.

### 5. Continuity

PiGenus runs continuously, reliably, and resource-efficiently:

- 24/7 availability
- Low power usage
- Automatic recovery
- Regular maintenance
- Scheduled background work
- Long-term stability

PiGenus embodies continuity.

## What PiGenus Is Not

PiGenus is not:

- The main LLM
- A GPU cluster
- A pure frontend
- A throwaway project
- A Raspberry Pi toy experiment
- An autonomous end in itself

## Relationship To GENUS

GENUS is the overall system of intelligent capabilities and experiences. PiGenus is its local reliable core.

Working comparison:

- GENUS: organism
- PiGenus: memory, coordination layer, circulation, and operations desk
- Workers: muscles
- Models: thinking tools
- Clients: eyes, hands, and voice

## Operating Philosophy

PiGenus follows these principles:

- Not maximum performance, but maximum reliability.
- Not doing everything itself, but coordinating everything sensibly.
- Not short-term impressiveness, but long-term support.
- Reliable private orchestration node over flashy AI gimmick.

## Operating Modes

### Real-Time Mode

- Accept requests
- Coordinate jobs
- Deliver status
- Mediate responses
- Check security

### Background Mode

- Condense memory
- Create backups
- Clean logs
- Prepare tasks
- Synchronize data
- Generate reports
- Trigger optimizations

## Non-Negotiable Design Rule

Whenever a design decision is uncertain, choose the path that makes PiGenus a more reliable, private, recoverable orchestration and memory node.
